# 📚 1) 구조화된 정리본 (원문 최대 유지 + 시험 구조 재배열)

## 1. 빅데이터 개념과 정의

### 1-1. 빅데이터 정의
- 기존 데이터베이스 관리 도구로 처리 불가능한 데이터셋
- 수백 TB ~ PB 이상 규모 데이터
- 하루 약 250경 바이트 생성
- 데이터 생성 속도 연 40% 증가
- 2025년 약 181 ZB 규모 예상
### 1-2. 빅데이터 포함 범위

- 데이터 + 분석 기술 + 인력 + 조직 + 인프라 포함
- 단순 데이터가 아닌 “통합 시스템 개념”

### 1-3. 데이터 생성 특징

- 90% 데이터가 최근 3년 내 생성
- 폭발적 증가 추세

---

## 2. 빅데이터 3V 특성

### 2-1. Volume (규모)

- 테라바이트 ~ 페타바이트 이상
- 초대용량 데이터 생성

### 2-2. Velocity (속도)

- 실시간 데이터 생성 및 처리
- 빠른 의사결정 지원

### 2-3. Variety (다양성)

- 정형/반정형/비정형 데이터 포함
- 텍스트, 이미지, 영상, 음성 등

---

## 3. 데이터 유형

### 3-1. 정형 데이터

- RDBMS, 엑셀, 스프레드시트
- 구조 고정, 분석 용이

### 3-2. 반정형 데이터

- XML, HTML, 로그 데이터
- 구조 있지만 유연함

### 3-3. 비정형 데이터

- 이미지, 영상, 음성, SNS
- 전체 데이터의 80% 이상

---

## 4. 빅데이터 특징

### 4-1. 처리 특성

- 분산처리 필요
- 고성능 컴퓨팅 요구
- 기존 DB로 처리 불가

### 4-2. 분석 특성

- 비정형 데이터 중심
- 패턴 및 인사이트 발견
- 정확성 중심 분석

### 4-3. 전략적 가치

- 21세기 원유
- 자본·노동과 동등한 생산 요소

## 5. 데이터 자본의 특성

### 5-1. 비경쟁성

## 6. 빅데이터 가치

### 6-1. 투입가치

- 생산 요소 (자본/노동과 동일)
- 혁신 기반 요소
- 전략적 자산

### 6-2. 활용가치
