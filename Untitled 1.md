> **우주왕복선은 재사용되었지만 정비 비용과 구조 복잡성 때문에 경제적 재사용에 실패했고,

# 슬라이드 2. 재사용 개념의 역사와 우주왕복선의 한계

## 슬라이드 내용

### 재사용 개념의 역사

- **1920~30년대**: 치올콥스키
- **1950년대**: 폰 브라운
- **1981~2011**: 우주왕복선
- **1990년대**: DC-X
- **2015년**: Falcon 9 1단 착륙 성공

### 우주왕복선의 한계

- 부분 재사용만 가능
    - 본체, 고체부스터 재사용
    - 외부 연료탱크는 폐기
- 재진입 후 **수만 장의 방열 타일** 검사/교체
- 재발사 준비에 수개월 소요
- 유지보수 비용이 매우 큼
- 결과적으로 **발사 비용 감소 효과가 제한적**

### 발표 대본

재사용 로켓의 개념은 1920년대와 30년대부터 제안되었습니다.  
치올콥스키가 재사용 가능한 우주선 개념을 이야기했고, 1950년대에는 폰 브라운도 재사용 우주비행체를 구상했습니다. 

실제로 비행한 대표적 재사용 시스템은 우주왕복선입니다.  
하지만 우주왕복선은 완전한 의미의 경제적 재사용에는 실패했습니다. 

그 이유는 첫째, 부분 재사용이었기 때문입니다.  
본체와 고체부스터는 회수했지만 외부 연료탱크는 버렸습니다. 

둘째, 유지보수가 지나치게 복잡했습니다.  
재진입 후에는 수만 장의 방열 타일을 검사하고 필요한 부분을 교체해야 했고, 재발사 준비에 수개월이 걸렸습니다. 

즉, 우주왕복선은 재사용되기는 했지만, 정비 비용과 시간이 커서 ‘비행기처럼 빠르게 다시 쓰는 재사용’과는 거리가 있었습니다.

# 슬라이드 3. 왜 상용화까지 오래 걸렸는가

## 슬라이드 내용

### 상용화가 늦어진 이유

1. **제어 문제**
    - 자세, 위치, 속도를 동시에 제어
    - 질량과 공력이 계속 변화
2. **동체 재사용 문제**
    - Max Q 구조 하중
    - 재진입 열
    - 반복 피로와 정비
3. **엔진 신뢰성 문제**
    - 정확한 재점화
    - 추력 조절
    - 반복 사용

### 발표 대본

재사용 상용화가 늦어진 이유는 단순히 기술이 하나 부족해서가 아니었습니다. 

첫째는 제어 문제입니다.  
하강 중에는 위치, 속도, 자세를 동시에 맞춰야 하고, 연료가 줄면서 질량도 계속 바뀝니다. 공력도 속도와 자세에 따라 달라집니다. 

둘째는 동체 재사용 문제입니다.  
상승 중 Max Q에서 큰 하중을 받고, 하강 중 재진입 열을 받으며, 이것이 반복되면 피로가 누적됩니다. 

셋째는 엔진 문제입니다.  
착륙을 위해서는 엔진이 다시 켜져야 하고, 필요한 만큼만 추력을 내야 하며, 반복 사용에도 안정적이어야 합니다.

# 슬라이드 4. 제어 문제 ①: 왜 수직 착륙이 어려운가

## 슬라이드 내용

### 하강 운동

m\dot{v}=T-D-mg

### 자세 운동

I\dot{\omega}=M

- m: 질량
- v: 속도
- T: 추력
- D: 항력
- I: 관성모멘트
- \omega: 각속도
- M: 모멘트

### 어려운 이유

- 질량 m이 계속 감소
- 항력 D는 속도와 자세에 따라 변함
- 외란: 바람, 슬로싱, 센서 오차

### 발표 대본

수직 착륙은 단순히 아래로 내려오는 문제가 아닙니다.  
하강 방향 운동은

m\dot{v}=T-D-mg

로 나타낼 수 있습니다.  
즉, 속도 변화는 추력과 항력, 중력의 차이로 결정됩니다. 

또 자세는

I\dot{\omega}=M

로 나타낼 수 있습니다.  
즉, 회전은 모멘트에 의해 바뀝니다. 

문제는 여기서 질량 m이 연료 소모로 계속 줄어든다는 점입니다.  
항력 D도 속도와 자세, 공기 밀도에 따라 계속 바뀝니다.  
즉, 이 시스템은 시간이 지남에 따라 계수가 바뀌는 비선형 제어 문제입니다.

# 슬라이드 5. 제어 문제 ②: PID, LQR, 비선형 제어

## 슬라이드 내용

### PID 제어

u(t)=K_P e(t)+K_I \int e(t)\,dt + K_D \frac{de(t)}{dt}

- e(t): 목표값과 현재값의 오차
- 비례, 적분, 미분 제어의 합

### LQR 제어

J=\int_0^\infty (x^TQx + u^TRu)\,dtu=-Kx

- 상태 x와 제어입력 u의 비용을 최소화
- 오차와 제어 에너지를 동시에 고려

### 핵심

- 로켓 착륙은 단순 PID 하나로 끝나기 어려움
- 비선형 시스템을 구간별로 선형화하고 계속 업데이트해야 함

### 발표 대본

실제 제어 알고리즘을 보면, 가장 기본적인 형태는 PID입니다.

u(t)=K_P e(t)+K_I \int e(t)\,dt + K_D \frac{de(t)}{dt}

여기서 e(t)는 목표와 현재 상태의 차이입니다.  
비례항은 현재 오차에 반응하고, 적분항은 누적 오차를 줄이며, 미분항은 변화 속도에 반응합니다. 

하지만 로켓 착륙은 PID만으로 다루기 어려운 경우가 많습니다.  
이유는 상태가 계속 변하고, 제어해야 할 축이 여러 개이며, 연료와 공력 조건도 계속 바뀌기 때문입니다. 

그래서 더 고급 방식으로는 LQR 같은 상태공간 제어가 사용됩니다.

J=\int_0^\infty (x^TQx + u^TRu)\,dt

이 식은 상태 오차와 제어 입력을 동시에 최소화하는 비용함수입니다.  
그 결과 제어입력은 보통

u=-Kx

형태가 됩니다. 

중요한 점은, 로켓은 완전한 선형 시스템이 아니므로 하나의 고정된 제어기만으로 다루기 어렵다는 것입니다.  
실제로는 구간별로 선형화한 모델, lookup table, 상태추정, 센서 피드백을 계속 업데이트하며 제어합니다.

# 슬라이드 6. 제어 문제 ③: CFD, lookup table, belly flop, grid fin, sloshing

## 슬라이드 내용

### CFD → lookup table

- 공력 계수 C_D, C_L, C_M를 오프라인 계산
- 비행 중 실시간 CFD는 수행하지 않음
- 센서값으로 현재 조건에 맞는 계수 선택

### 제어 수단

- Falcon 9: grid fin, cold gas thruster
- Starship: belly flop, 플랩, flip maneuver
- 슬로싱: 무게중심 이동 → 제어 불안정

### 발표 대본

공력 제어 특성은 사전에 CFD로 계산합니다.  
예를 들어 항력 계수 C_D, 양력 계수 C_L, 모멘트 계수 C_M를 고도와 속도, 자세에 따라 계산해 둡니다.  
비행 중에는 정밀 CFD를 다시 돌리지 않고, lookup table에서 현재 조건에 맞는 값을 가져와 사용합니다. 

Falcon 9은 그리드 핀과 콜드 가스 스러스터를 이용해 하강 중 자세를 잡습니다.  
Starship은 belly flop으로 공기 저항을 키우고, 플랩과 flip maneuver로 착륙 자세를 만듭니다. 

여기에 슬로싱까지 더해지면 탱크 안 연료가 출렁이며 무게중심이 바뀝니다.  
즉, 제어 대상 자체가 계속 변하는 시스템이 됩니다.

# 슬라이드 7. 동체 재사용 문제 ①: Max Q와 구조 하중

## 슬라이드 내용

q=\frac{1}{2}\rho v^2

- Max Q: 동압 최대
- 발사 후 약 **1분**
- 고도 **10~15 km**
- 동압 약 **30~50 kPa**

### 의미

- 공력 하중 최대
- 동체, 핀, 인터스테이지에 큰 응력
- throttle down 필요

### 발표 대본

동체 재사용에서 첫 번째 문제는 구조 하중입니다.  
대표적인 구간은 Max Q입니다. 

q=\frac{1}{2}\rho v^2

이 식에서 보듯이 동압은 공기 밀도와 속도의 제곱에 비례합니다.  
보통 발사 후 약 1분, 고도 10에서 15킬로미터 사이에서 Max Q가 나타나며, 동압은 약 30에서 50킬로파스칼 수준입니다. 

이 구간에서는 동체와 핀, 인터스테이지가 가장 큰 공력 하중을 받기 때문에, 엔진 출력을 잠시 낮춰 하중을 줄입니다.

# 슬라이드 8. 동체 재사용 문제 ②: 재진입 열과 열피로

## 슬라이드 내용

### 열 발생 원인

- 공기 압축에 의한 단열 가열
- 대류열 + 복사열

### 구조 영향

- 열팽창
- 열응력
- 열피로
- 고온 강도 저하

### 수치

- 표면/열차폐: **수백~1000°C 이상급**
- 엔진 연소 가스: **약 3000°C 전후**

### 발표 대본

두 번째는 열 문제입니다.  
재진입 열은 단순 마찰이 아니라, 공기가 압축되며 가열되는 단열 가열이 핵심입니다.  
이 열은 대류와 복사를 통해 구조물에 전달됩니다. 

구조 입장에서는 열팽창이 생기고, 연결된 부위에는 열응력이 발생합니다.  
이 과정이 반복되면 열피로가 누적되고, 재료 강도도 낮아집니다. 

수치로 보면 재진입 표면이나 열차폐는 수백 도에서 1000도 이상급 환경을 견뎌야 하고, 엔진 연소 가스는 약 3000도 전후입니다.

# 슬라이드 9. 동체 재사용 문제 ③: 정비, 데이터 기반 손상 추정, 재료

## 슬라이드 내용

### 왜 정비가 중요한가

- 착륙 성공 ≠ 바로 재사용 가능
- 누적 피로, 미세균열, 열손상 확인 필요

### 방법

- CFD: 열유속·압력 분포
- FEA:Ku=F
- 비행 이력 + 검사 데이터로 손상 추정

### 재료

- 티타늄 그리드 핀
- TPS
- 엔진부 열차폐
- 생산 통합 → 빠른 설계 수정, 품질 관리

### 발표 대본

재사용의 핵심은 착륙 장면보다, 다음 비행에 써도 되는가를 판단하는 데 있습니다.  
즉, 정비와 손상 추정이 중요합니다. 

공력과 열하중은 CFD로 계산하고, 구조응답은 FEA로 계산합니다.  
FEA의 기본 형태는

Ku=F

로 쓸 수 있는데, 힘 F가 주어졌을 때 구조 변위 u를 계산하는 식입니다. 

이 결과와 실제 비행 이력, 검사 데이터를 결합해 어디에 손상이 누적되었는지 추정합니다.  
재료 측면에서는 티타늄 그리드 핀, 열보호시스템, 엔진부 열차폐가 대표적입니다.  
또한 SpaceX는 생산을 많이 내부화해 설계 변경과 품질 관리를 빠르게 수행했습니다.

# 슬라이드 10. 로켓 엔진과 신뢰성

## 슬라이드 내용

### 엔진이 필요한 이유

- 역추진 감속
- 최종 속도 제어
- 자세 보정

### 요구 성능

- 재점화
- 추력 조절
- 짐벌 제어
- 반복 사용 신뢰성

### 발표 대본

엔진은 재사용 로켓에서 추진기관인 동시에 착륙 제어 장치입니다.  
역추진 감속을 해야 하고, 마지막에는 속도를 거의 0에 맞춰야 합니다.  
또 필요하면 자세도 보정해야 합니다. 

그래서 엔진에는 재점화 능력, 추력 조절 능력, 짐벌 제어, 반복 사용 신뢰성이 모두 필요합니다.  
제어 알고리즘이 맞아도 엔진이 제때 켜지지 않으면 착륙은 실패합니다.

# 슬라이드 11. 결론

## 슬라이드 내용

### 결론

- 재사용 개념은 오래전부터 존재
- 우주왕복선은 재사용되었지만 정비 비용이 커서 경제성이 낮았음
- 수직 착륙 상용화가 늦어진 이유:
    - 비선형 제어 문제
    - 구조·열·피로 문제
    - 엔진 신뢰성 문제
- SpaceX의 돌파:
    - CFD/FEA 기반 설계
    - PID/LQR 계열 제어와 상태 업데이트
    - 데이터 기반 정비
    - 빠른 반복 개발

### 최종 문장

> 재사용 로켓은 단순한 착륙 기술이 아니라,  
> **비선형 제어와 구조·열·재료 문제를 통합한 반복 운용 시스템**입니다.

### 발표 대본

정리하면, 재사용 로켓의 개념은 오래전부터 존재했지만 상용화는 늦었습니다.  
우주왕복선은 재사용되었지만 정비 비용이 커서 경제성이 낮았습니다. 

수직 착륙 상용화가 늦어진 이유는 로켓이 비선형적이고 시간에 따라 변하는 제어 대상이었고, 동시에 구조 하중과 재진입 열, 피로, 엔진 신뢰성 문제까지 함께 해결해야 했기 때문입니다. 

SpaceX는 CFD와 FEA로 설계를 보완하고, PID와 LQR 계열 제어, 상태 업데이트, 비행 데이터 기반 정비, 빠른 반복 개발을 통해 이를 상용화했습니다. 

따라서 재사용 로켓은 단순한 착륙 기술이 아니라, 비선형 제어와 구조·열·재료 문제를 통합한 반복 운용 시스템이라고 결론 내릴 수 있습니다.

# 추가 발표 팁: PID/LQR를 어떻게 말해야 자연스러운가

발표에서 너무 수학적으로 보이지 않게 하려면 이렇게 말하면 됩니다.

### PID 설명 한 줄

> PID는 현재 오차, 누적 오차, 오차 변화 속도를 동시에 사용해 제어 입력을 만드는 기본 제어기입니다.

### LQR 설명 한 줄

> LQR는 상태 오차를 줄이면서도 제어 입력을 과도하게 쓰지 않도록 최적화하는 상태공간 제어 방법입니다.

### 비선형 문제 강조 한 줄

> 하지만 로켓은 질량, 공력, 외란이 계속 바뀌기 때문에, 하나의 고정된 선형 제어기로 끝나지 않고 모델과 상태를 계속 업데이트해야 합니다.

# 슬라이드 1. 도입: 재사용은 왜 중요한가

## 슬라이드 내용

- 기존 로켓 = 발사 후 폐기
- 재사용 로켓 = 회수 후 다시 사용
- 목표:
    - 발사 비용 절감
    - 발사 빈도 증가
    - 우주 접근성 확대
- **실질적 상용화 성공:** Falcon 9 1단 재사용

### 발표 대본

안녕하세요. 저는 오늘 로켓 수직 착륙과 재사용 기술에 대해 발표하겠습니다.  
기존 로켓은 한 번 발사하고 버리는 구조였습니다. 이는 비행기 한 대를 한 번 날리고 폐기하는 것과 비슷합니다.  
반면 재사용 로켓은 발사에 사용한 추진체를 회수해 다시 사용하는 기술입니다.  
이 기술이 중요한 이유는 비용을 줄이고 발사 빈도를 높일 수 있기 때문입니다. 

실제로 SpaceX의 Falcon 9은 1단 추진체를 반복적으로 회수해 다시 사용하면서, 재사용 로켓을 개념이 아니라 상용 시스템으로 만들었습니다.

# 슬라이드 2. 재사용 개념은 언제 등장했는가

## 슬라이드 내용

- **1920~30년대:** 치올콥스키 
    - 재사용 가능한 우주선 개념 제안
- **1950년대:** 베르너 폰 브라운 
    - 재사용 우주비행체 구상
- **1981~2011:** 우주왕복선
- **1990년대:** DC-X 수직이착륙 시험
- **2015년:** Falcon 9 1단 육상 착륙 성공

### 발표 대본

재사용 로켓의 개념은 최근에 나온 것이 아닙니다.  
(수식어) 폰 브라운은 이미 50년대에 (저서)에서  재사용 우주비행체 개념을 제시.

이후 실제 시도로는 우주왕복선이 있었고, 1990년대에는 NASA와 미 국방부가 DC-X라는 수직이착륙 시험선을 운영했습니다.  
하지만 실질적인 상용화는 훨씬 늦었습니다.  
Falcon 9이 처음 1단을 지상에 착륙시킨 것은 2015년입니다. 

즉, 개념 제안부터 상용화까지는 거의 수십 년의 시간차가 있었습니다.

# 슬라이드 3. 왜 상용화까지 오래 걸렸는가

## 슬라이드 내용

### 재사용이 어려운 이유

1. **제어 문제**
    - 하강 중 자세 유지
    - 착륙 지점 정밀 제어
    - 슬로싱, 바람, 외란
2. **동체 재사용 문제**
    - 구조 하중
    - 재진입 열
    - 반복 피로와 정비
3. **엔진 신뢰성 문제**
    - 재점화
    - 추력 조절
    - 정확한 타이밍

### 발표 대본

상용화가 오래 걸린 이유는 로켓을 다시 내려오게 하는 것이 단순한 역추진 문제가 아니기 때문입니다.  
크게 세 가지 문제가 있습니다. 

첫째는 제어 문제입니다.  
로켓은 길고 가는 구조라 하강 중 자세를 유지하기 어렵고, 바람이나 연료 출렁임의 영향도 큽니다. 

둘째는 동체 재사용 문제입니다.  
상승 중 Max Q에서 큰 구조 하중을 받고, 하강 중에는 재진입 열을 받습니다. 이것이 반복되면 피로가 누적됩니다. 

셋째는 엔진 문제입니다.  
착륙을 위해서는 엔진이 정확한 순간에 다시 켜져야 하고, 추력도 정밀하게 조절되어야 합니다. 

이 세 가지가 동시에 해결되어야 상용화가 가능합니다.

# 슬라이드 4. 제어 문제 ①: 왜 수직 착륙이 어려운가

## 슬라이드 내용

### 수직 착륙의 본질

- 길고 가는 강체의 불안정 제어
- 하강 중 속도: 수백 m/s 이상 → 착륙 직전 0 m/s 근처
- 자세, 위치, 속도를 동시에 제어해야 함

### 기본 운동 방정식

m\dot{v}=T-D-mg

- m: 질량
- T: 추력
- D: 항력
- mg: 중력

### 발표 대본

수직 착륙이 어려운 이유는 로켓이 공력적으로 안정적인 비행체가 아니기 때문입니다.  
비유하면 긴 막대를 세운 채로 움직이면서 정확한 위치에 내려놓는 것과 비슷합니다. 

기본적으로 하강 방향 운동은 

m\dot{v}=T-D-mg

로 나타낼 수 있습니다.  
즉, 속도 변화는 추력, 항력, 중력의 차이로 결정됩니다. 

착륙에서는 하강 속도가 수백 미터 퍼 세컨드에서 최종적으로 0에 가까워져야 합니다.  
그런데 동시에 위치와 자세도 맞춰야 하므로, 단순히 엔진을 켜는 것만으로는 부족합니다.

# 슬라이드 5. 제어 문제 ②: CFD, lookup table, 제어 알고리즘

## 슬라이드 내용

### 설계 단계

- CFD로 공력 계수 계산
- 예: C_D, C_L, C_M

### 비행 중

- CFD를 실시간으로 돌리지 않음
- 미리 계산한 공력 데이터베이스 사용
- lookup table + 센서 피드백
- PID / LQR / 상태추정 기반 제어

### 동압

q=\frac{1}{2}\rho v^2

### 발표 대본

하강 제어에서 중요한 것은 공력 특성을 미리 아는 것입니다.  
설계 단계에서는 CFD, 즉 전산유체역학으로 로켓의 항력 계수 C_D, 양력 계수 C_L, 모멘트 계수 C_M를 계산합니다. 

하지만 실제 비행 중에는 CFD를 다시 계산하지 않습니다.  
정밀 CFD는 계산량이 너무 크기 때문입니다. 

대신 미리 계산한 데이터를 표 형태로 저장해 두고, 현재 속도와 자세에 맞는 값을 lookup table에서 꺼내 사용합니다.  
그리고 센서 데이터와 함께 PID나 LQR 같은 제어 알고리즘으로 자세를 수정합니다. 

여기서 중요한 값이 동압입니다.

q=\frac{1}{2}\rho v^2

공기 밀도와 속도가 클수록 공력 하중이 커진다는 의미입니다.

# 슬라이드 6. 제어 문제 ③: belly flop, grid fin, sloshing

## 슬라이드 내용

### Falcon 9

- **Grid fin**
    - 하강 중 공력 제어
- **Cold gas thruster**
    - 진공 구간 자세 제어

### Starship

- **Belly flop**
    - 압력항력 증가
- **플랩 제어**
    - 무게중심(CG)과 압력중심(CP) 관계 조절

### 슬로싱

- 연료가 출렁이며 무게중심 이동
- 제어 안정성 악화

### 발표 대본

제어 문제를 실제 기체에 적용한 예를 보면, Falcon 9은 그리드 핀을 사용합니다.  
그리드 핀은 재진입 중 공기 흐름을 이용해 방향을 바꾸는 장치입니다.  
대기권 밖에서는 콜드 가스 스러스터로 자세를 맞춥니다. 

Starship은 더 독특합니다.  
배면 낙하, 즉 belly flop을 사용해 넓은 면적으로 공기 저항을 받아 감속합니다.  
이때 플랩을 조절해 압력중심, 즉 CP의 위치를 바꿔 자세를 안정화합니다. 

또 하나 중요한 문제가 슬로싱입니다.  
착륙 직전 남은 연료가 탱크 안에서 출렁이면 무게중심 CG가 이동해 제어가 어려워집니다.  
Starship의 헤더 탱크는 이 문제를 줄이기 위한 설계입니다.

# 슬라이드 7. 동체 재사용 문제 ①: Max Q와 구조 하중

## 슬라이드 내용

### Max Q

q=\frac{1}{2}\rho v^2

- 발사 후 약 **1분 전후**
- 고도 약 **10~15 km**
- 동압 약 **30~50 kPa 수준**

### 의미

- 구조물에 가장 큰 공력 하중
- throttle down 필요

### 발표 대본

동체 재사용 문제의 첫 번째는 구조 하중입니다.  
가장 대표적인 구간이 Max Q입니다. 

Max Q는 동압이 최대가 되는 순간이고, 수식은 다시

q=\frac{1}{2}\rho v^2

입니다. 

대부분의 로켓은 발사 후 약 1분 전후, 고도 10에서 15킬로미터 부근에서 Max Q를 겪습니다.  
이때 동압은 대략 30에서 50킬로파스칼 수준입니다. 

중요한 점은 로켓이 가장 빠를 때가 아니라, 공기가 아직 충분히 남아 있는데 속도도 커진 구간이 가장 위험하다는 것입니다.  
그래서 실제 발사에서는 이 구간에서 엔진 출력을 낮춰 구조 하중을 줄입니다.

# 슬라이드 8. 동체 재사용 문제 ②: 재진입 열과 열피로

## 슬라이드 내용

### 재진입 열의 원인

- 단순 마찰이 아니라 **공기 압축**
- 운동에너지 → 내부에너지 → 열

### 열 관련 문제

- 열팽창
- 열응력
- 열피로
- 고온 강도 저하

### 대표 수치

- 표면/열차폐 환경: **수백~1000°C 이상급**
- 엔진 연소 가스: **약 3000°C 전후**

### 발표 대본

두 번째는 열 문제입니다.  
재진입 열은 흔히 마찰열이라고 표현되지만, 실제로는 공기가 압축되며 가열되는 효과가 더 중요합니다.  
즉, 로켓의 운동에너지가 공기의 내부에너지와 열로 바뀌는 과정입니다. 

이 열은 동체에 여러 문제를 만듭니다.  
온도 차이로 인해 부위별 열팽창이 다르게 일어나고, 연결된 구조 내부에는 열응력이 생깁니다.  
이 과정이 반복되면 열피로가 누적되고, 재료 강도도 낮아집니다. 

대표적으로 재진입 시 표면이나 열차폐 시스템은 수백 도에서 1000도 이상급 환경을 견뎌야 하고, 엔진 내부 연소 가스는 약 3000도 전후입니다.

# 슬라이드 9. 동체 재사용 문제 ③: 데이터 기반 손상 추정·정비·재료

## 슬라이드 내용

### 왜 모델이 중요한가

- 겉보기 멀쩡해도 내부 손상 가능
- 반복 사용 → 피로 누적 추정 필요

### 사용 방법

- CFD: 열유속, 압력 분포
- FEA: 응력, 변형, 피로 수명
- 비행 데이터와 결합해 정비 우선순위 결정

### 재료 사례

- Falcon 9 Block 5:
    - **티타늄 grid fin**
    - **TPS**
    - **엔진부 열차폐**

### 발표 대본

재사용에서 중요한 것은 착륙 성공 자체보다, 다음 비행에도 안전하게 쓸 수 있는가입니다.  
겉보기에는 멀쩡해 보여도 내부에는 열피로나 미세 균열이 누적될 수 있습니다. 

그래서 모델이 중요합니다.  
CFD로 어느 부위가 가장 뜨거웠는지, 어느 부위에 압력이 집중됐는지 계산하고,  
FEA로 응력과 변형, 피로 수명을 예측합니다.  
그리고 실제 비행 데이터를 결합해 어느 부위를 우선 점검할지 정합니다. 

재료 측면에서 Falcon 9 Block 5는 티타늄 그리드 핀, 열보호 시스템, 엔진부 열차폐를 적용해 반복 사용에 대응했습니다.

# 슬라이드 10. 로켓 엔진과 신뢰성

## 슬라이드 내용

### 엔진의 역할

- 역추진 감속
- 자세 수정
- 최종 착륙 속도 제어

### 요구 조건

- 정확한 재점화
- 추력 조절
- 짐벌 제어
- 반복 사용 신뢰성

### 핵심

- 제어가 맞아도 엔진이 실패하면 착륙 실패

### 발표 대본

마지막 핵심은 엔진입니다.  
제어 알고리즘이 아무리 좋아도 엔진이 제때 켜지지 않으면 착륙은 실패합니다. 

착륙 엔진은 정확한 타이밍에 재점화되어야 하고, 추력을 조절할 수 있어야 하며, 방향도 제어할 수 있어야 합니다.  
또한 한 번만 작동하는 것이 아니라 반복적으로 사용 가능해야 합니다. 

즉, 재사용 로켓 엔진은 단순 추진 장치가 아니라, 감속기이자 자세 제어기라고 볼 수 있습니다.

# 슬라이드 11. 결론

## 슬라이드 내용

### 결론

- 재사용 개념은 오래전부터 존재
- 상용화가 늦었던 이유:
    - 제어 문제
    - 구조·열 문제
    - 엔진 신뢰성
- SpaceX의 돌파:
    - CFD/FEA 기반 설계
    - lookup table + GNC 제어
    - 데이터 기반 정비
    - 빠른 반복 개발

### 최종 문장

> 재사용 로켓의 핵심은 단순히 다시 착륙하는 것이 아니라,  
> **공력·열·구조·제어를 통합해 반복 운용 가능한 시스템으로 만드는 것**입니다.

### 발표 대본

정리하면, 재사용 로켓은 오래전부터 제안되었지만 상용화까지 오래 걸렸습니다.  
그 이유는 단순히 엔진 출력이 부족해서가 아니라, 제어 문제, 구조와 열 문제, 엔진 신뢰성 문제가 동시에 존재했기 때문입니다. 

SpaceX는 CFD와 FEA로 설계 단계의 문제를 줄이고, 실제 비행에서는 lookup table과 GNC 제어를 사용했으며, 비행 데이터 기반 정비와 빠른 반복 개발을 통해 상용화를 이뤘습니다. 

결국 재사용 로켓의 핵심은 다시 착륙하는 장면 자체가 아니라, 공력과 열, 구조와 제어를 통합해 반복 운용 가능한 시스템으로 만드는 데 있다고 결론 내릴 수 있습니다.

# 발표에 넣을 수식 정리

발표에서 실제로 넣을 수식은 **2개면 충분**합니다.

## 1) 동압

q=\frac{1}{2}\rho v^2

- Max Q 설명용
- 구조 하중 설명용

## 2) 수직 방향 운동

m\dot{v}=T-D-mg

- 추력, 항력, 중력의 균형
- 착륙 감속 설명용

추가로 1줄 설명 정도로:

- C_D, C_L, C_M: CFD로 미리 계산되는 공력 계수
- PID / LQR: 비행 중 제어 알고리즘

이 정도가 10분 발표에 가장 적절합니다.
